SOFTWARE

El software es el conjunto de programas, instrucciones y datos que permiten el funcionamiento de una computadora. Se compone de varias partes y se divide en distintas categorías según su función y estructura. 1. Componentes del software El software se compone de tres elementos principales: a) Código fuente Conjunto de instrucciones escritas en un lenguaje de programación. Puede ser compilado o interpretado para ejecutarse en la computadora. Ejemplo: Un programa escrito en C++ o Python. b) Datos Información que el software usa para funcionar. Pueden ser archivos de texto, bases de datos, imágenes, etc. Ejemplo: Una base de datos en MySQL o archivos de configuración. c) Documentación Manuales, guías y descripciones del software. Ayuda a programadores y usuarios a entender su uso y funcionamiento. Ejemplo: Documentación técnica de un software o tutoriales de usuario. 2. División del software El software se divide en varias categorías según su propósito: A) Software de sistema Controla el hardware y permite la ejecución de otros programas. Incluye: Sistemas operativos (Windows, Linux, macOS). Controladores de dispositivos (drivers de impresora, tarjeta gráfica). Utilidades del sistema (antivirus, herramientas de diagnóstico). B) Software de aplicación Programas diseñados para que el usuario realice tareas específicas. Incluye: Aplicaciones de oficina (Word, Excel). Navegadores web (Chrome, Firefox). Juegos y programas de entretenimiento. C) Software de programación Herramientas que permiten desarrollar otros programas. Incluye: Editores de código (VS Code, Sublime Text). Compiladores e intérpretes (GCC, Python). Entornos de desarrollo (IDE) (Eclipse, Visual Studio). D) Software de middleware Actúa como intermediario entre diferentes aplicaciones o sistemas. Se usa en redes y bases de datos para conectar distintos programas. Ejemplo: Un servidor de aplicaciones como Apache Tomcat..

Software de Sistema (System Software)

Es el software que gestiona los recursos del hardware y proporciona servicios básicos para otros programas. Se ejecuta en segundo plano y es esencial para el funcionamiento de la computadora. Características del Software de Sistema Actúa como intermediario entre el hardware y el usuario. Permite la ejecución de otros programas. Funciona en segundo plano sin intervención constante del usuario. Generalmente desarrollado en lenguajes de bajo nivel (C, C++, ensamblador). Tipos de Software de Sistema a) Sistemas Operativos (Operating Systems - OS) Son el núcleo del software de sistema, ya que administran el hardware y proporcionan una interfaz para los usuarios. Funciones: Gestión de memoria y procesos. Administración de archivos y dispositivos. Control de usuarios y seguridad. Ejemplos: Windows, macOS, Linux, Android, iOS. b) Controladores de Dispositivos (Device Drivers) Son programas que permiten la comunicación entre el sistema operativo y el hardware específico. Funciones: Hacer que los dispositivos de hardware funcionen correctamente. Traducir las instrucciones del sistema operativo en comandos para el hardware. Ejemplos: Driver de la tarjeta gráfica (NVIDIA, AMD). Driver de la impresora (HP, Canon). c) Software de Utilidades (Utility Software) Son programas diseñados para mantener, optimizar o proteger el sistema. Funciones: Optimización del rendimiento del sistema. Protección contra virus y malware. Mantenimiento de archivos y discos duros. Ejemplos: Antivirus (Avast, Norton). Software de limpieza (CCleaner). Herramientas de diagnóstico del sistema. 2. Software de Aplicación (Application Software) Es el software diseñado para realizar tareas específicas de los usuarios. A diferencia del software de sistema, este no es esencial para el funcionamiento de la computadora, pero es clave para la productividad y el entretenimiento. Características del Software de Aplicación Diseñado para usuarios finales. Se enfoca en tareas específicas. Generalmente tiene una interfaz gráfica amigable. Puede ser instalado y desinstalado según las necesidades del usuario. Tipos de Software de Aplicación a) Software de Productividad Facilita tareas como edición de textos, gestión de datos y diseño. Ejemplos: Microsoft Office (Word, Excel, PowerPoint). Google Docs, Sheets y Slides. AutoCAD (diseño asistido por computadora). b) Software de Entretenimiento y Multimedia Incluye programas para ver videos, escuchar música o editar contenido. Ejemplos: VLC Media Player. Adobe Photoshop (edición de imágenes). Spotify, Netflix. c) Software de Comunicación Permite la comunicación entre usuarios o dispositivos. Ejemplos: WhatsApp, Telegram, Zoom. Correos electrónicos (Outlook, Gmail). d) Software de Juegos y Simulación Programas diseñados para el entretenimiento o el aprendizaje a través de la simulación. Ejemplos: Juegos de PC y consolas (Minecraft, GTA, FIFA). Simuladores de vuelo o conducción (Microsoft Flight Simulator). e) Software de Gestión Empresarial Ayuda en la administración de negocios y procesos empresariales. Ejemplos: SAP (gestión de recursos empresariales). QuickBooks (contabilidad). CRM como Salesforce.

Arquitectura del software (estructura y capas)

La arquitectura del software es el diseño o la estructura de un programa, definiendo cómo se organizan sus partes y cómo interactúan entre sí. Estructura en capas (modelo común en aplicaciones): Capa de presentación: Es la interfaz de usuario, lo que ves en la pantalla (botones, menús). Ejemplo: La pantalla de inicio de una app bancaria. Capa lógica (negocio): Procesa la información y aplica reglas. Ejemplo: Cuando envías dinero en la app, esta verifica si tienes saldo suficiente. Capa de datos: Maneja la base de datos donde se guarda la información. Ejemplo: Guardar el saldo actualizado después de una transferencia. Otras arquitecturas: Monolítica: Todo el software está en un solo bloque. Cliente-servidor: Una parte del software está en el usuario (cliente) y otra en un servidor remoto. Microservicios: El software está dividido en pequeños módulos independientes. 2. Algoritmos y estructuras de datos (procesamiento eficiente) Un algoritmo es un conjunto de pasos para resolver un problema. Una estructura de datos es la forma en que organizamos la información en un programa. Ejemplo de algoritmo: Para ordenar una lista de números, un algoritmo como "Bubble Sort" compara y mueve los números hasta que están en orden. Ejemplo de estructura de datos: Listas: Una fila de tareas pendientes. Pilas (stack): Como una pila de platos; el último que entra es el primero que sale. Colas (queue): Como una fila de espera en el banco; el primero que entra es el primero que sale. Las estructuras de datos bien diseñadas hacen que los programas sean más rápidos y eficientes. 3. Seguridad del software (protección contra ataques) La seguridad evita que hackers roben o dañen información en el software. Principales amenazas: Malware: Virus que dañan el sistema. Ataques de hackers: Robar información con técnicas como el "phishing". Cómo se protege un software: Cifrado: Convertir datos en un código secreto (ejemplo: el cifrado de WhatsApp). Firewalls: Barreras que bloquean accesos no autorizados. Autenticación fuerte: Uso de contraseñas seguras y verificación en dos pasos. 4. Pruebas de software (evitar errores antes de lanzar el programa) Antes de que un software se use, debe ser probado para detectar fallos. Tipos de pruebas: Pruebas unitarias: Revisan pequeñas partes del código individualmente. Pruebas de integración: Verifican que todos los módulos del software trabajen bien juntos. Pruebas de carga: Simulan a muchos usuarios usando el software al mismo tiempo. Ejemplo: Antes de lanzar una app de pagos, los desarrolladores prueban qué pasa si alguien intenta enviar dinero sin saldo. 5. Mantenimiento y actualizaciones (mejoras constantes) Después de lanzar un software, hay que actualizarlo para corregir errores o agregar funciones nuevas. Tipos de mantenimiento: Correctivo: Solucionar errores o fallos detectados. Evolutivo: Agregar nuevas funciones. Preventivo: Mejorar la seguridad y optimización antes de que haya problemas. Ejemplo: WhatsApp lanza actualizaciones para corregir fallos y añadir nuevas funciones, como los stickers animados. 6. APIs (interacción con otros sistemas) Una API (Interfaz de Programación de Aplicaciones) permite que un software se comunique con otro. Ejemplo: Cuando un sitio web usa Google Maps para mostrarte la ubicación de una tienda, está usando la API de Google Maps. Una app de comida como PedidosYa usa la API de PayPal para procesar pagos. Las APIs hacen que los programas sean más flexibles y puedan conectarse con otros servicios sin necesidad de reinventar todo el código. 7. Modelos de desarrollo (formas de crear software) Existen diferentes métodos para desarrollar software. Principales modelos: Modelo en cascada (Waterfall): Se sigue una secuencia fija: Análisis → Diseño → Codificación → Pruebas → Implementación. Ventaja: Estructurado y claro. Desventaja: Si hay un error, regresar a una fase anterior es difícil. Modelo ágil (Agile): Se desarrolla en pequeñas partes llamadas "sprints". Permite mejorar el software constantemente con la retroalimentación del usuario. Ejemplo: Apps como Instagram actualizan funciones cada pocas semanas. Modelo DevOps: Integra desarrollo (Dev) y operaciones (Ops) para automatizar pruebas y lanzamientos. Se usa en grandes proyectos donde se necesita estabilidad y rapidez.

Requisitos del Software

Antes de desarrollar un software, es importante definir sus objetivos y funciones. Requisitos funcionales: Qué debe hacer el software. Ejemplo: Una app de mensajería debe permitir enviar mensajes y archivos. Requisitos no funcionales: Cómo debe hacerlo. Ejemplo: La app debe ser rápida y segura. Si no se definen bien estos requisitos, el software puede tener problemas más adelante. 2. Experiencia de Usuario (UX) e Interfaz de Usuario (UI) Un software debe ser fácil de usar para que los usuarios lo adopten rápidamente. Experiencia de Usuario (UX): Cómo se siente usar la aplicación. Ejemplo: Un sitio web debe cargar rápido y ser fácil de navegar. Interfaz de Usuario (UI): Diseño visual de botones, colores y textos. Ejemplo: Los íconos de una app deben ser intuitivos. Un mal diseño puede hacer que la gente deje de usar un software, aunque funcione bien técnicamente. 3. Rendimiento y Eficiencia del Software Un software debe funcionar rápido y consumir pocos recursos. Optimización de código: Evitar funciones innecesarias o lentas. Uso de memoria: Asegurar que el programa no ocupe demasiada RAM. Tiempo de respuesta: Reducir el tiempo que tarda el software en ejecutar tareas. Ejemplo: Un videojuego que se traba o se demora en cargar necesita optimización. 4. Compatibilidad y Portabilidad El software debe funcionar en diferentes dispositivos y sistemas operativos. Compatibilidad: Un programa debe funcionar bien en distintos sistemas. Ejemplo: WhatsApp funciona en Android, iOS y Windows. Portabilidad: Un software puede moverse fácilmente de un entorno a otro. Ejemplo: Un documento de Word se puede abrir en otro sistema sin problemas. 5. Escalabilidad del Software Es la capacidad del software para manejar más usuarios o más datos sin fallar. Escalabilidad vertical: Mejorar el hardware para soportar más carga. Ejemplo: Un servidor más potente para soportar más usuarios en un juego en línea. Escalabilidad horizontal: Agregar más servidores o procesos para distribuir la carga. Ejemplo: Un sitio web como YouTube usa múltiples servidores para manejar millones de usuarios. Si un software no es escalable, se volverá lento cuando tenga muchos usuarios. 6. Documentación del Software Escribir documentos que expliquen cómo funciona el software ayuda a programadores y usuarios. Documentación técnica: Explica el código y la arquitectura. Ejemplo: Manual para desarrolladores que quieran mejorar el software. Documentación para usuarios: Explica cómo usar el software. Ejemplo: Tutoriales de Microsoft Office. Sin documentación, es difícil actualizar o reparar un software en el futuro. 7. Gestión de Versiones y Control de Código El software cambia con el tiempo, por lo que se necesita un sistema para guardar versiones anteriores y coordinar el trabajo de los programadores. Git y GitHub/GitLab: Permiten manejar versiones del código y colaborar en equipo. Versionado de software: Se usan números como v1.0, v2.0 para indicar cambios. Ejemplo: Android 13 es una nueva versión del sistema operativo con mejoras respecto a Android 12. Si no se gestiona bien el código, pueden surgir errores y pérdida de datos importantes. 8. Licencias de Software Define cómo se puede usar y distribuir un software. Software propietario: No se puede modificar ni distribuir sin permiso. Ejemplo: Windows, Microsoft Office. Software libre (Open Source): El código es público y modificable. Ejemplo: Linux, Firefox. Software de pago y gratuito: Algunos programas son gratis, otros requieren licencia o suscripción. Elegir la licencia correcta es importante para proteger los derechos del software.
